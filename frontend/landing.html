How BusTrack works

01
Driver starts trip
The bus driver opens BusTrack and starts their route. GPS data begins streaming to the backend every 2 seconds.
02
Backend processes data
FastAPI receives GPS pings, saves them to SQLite, calculates ETAs using Haversine formula, and broadcasts via WebSocket.
03
Students see live map
The dashboard updates in real time. Students see bus position, speed, occupancy, and arrival time for their stop.
04
Resilience kicks in
If GPS or network drops, dead reckoning and offline caching keep everything running. Students never see an error.
Core innovation

Built to never fail

Most tracking systems show a blank screen when GPS or network fails. BusTrack uses three layers of resilience to keep the UI accurate and informative no matter what.

GPS Dead Reckoning
When GPS signal is lost, the frontend switches to a deterministic prediction engine using last known speed and heading.
dist = (speed/3600) × elapsed
new_lat = lat + (dist × cos(hdg)) / 111
WebSocket Store & Forward
Missed updates are cached in memory during disconnection. On reconnect, flushCache() replays them in order.
offlineCache[] → flushCache()
reconnect every 3s automatically
Haversine ETA
Great-circle distance formula gives accurate ETAs to every stop, even during predicted mode.
a = sin²(Δlat/2) + cos(lat1)·cos(lat2)·sin²(Δlng/2)
ETA = (R·2·atan2(√a,√(1-a))) / speed × 60
